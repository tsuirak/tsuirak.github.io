权限管理表

菜单 <- 多对多 -> 角色 <- 多对多 -> 用户

一个用户拥有若干角色，每一个角色拥有若干权限。

acl_permission

权限菜单表。

是用户可以访问的资源，包括页面权限、操作权限、数据权限等。

每个权限可以分配给多个角色。

acl_role

角色表。

角色是什么？可以理解为一定数量的权限的集合，权限的载体。每个角色可以被多个用户使用。每个角色可以拥有多个权限。

例如：一个论坛系统，超级管理员、版主，都是角色。版主可管理版内的帖子、可管理版内的用户等，这些是权限。要给某个用户授予这些权限，不需要直接将权限授予用户，可将 版主 这个角色赋予该用户。

acl_user

用户表。即参与后台管理的主体，每个用户都可以被分配多个角色。

acl_role_permission

角色和权限是多对多关系。

acl_user_role

用户和角色是多对多关系。

问题总结

  1. 路由切换问题

多次跳转到同一个路由页面，而 created 方法只执行一次，数据不会清空。

解决方法，使用watch 进行监听。

  1. mybatisplus 主键生成策略问题，（19 位 id值）

mp 生成的 id 值是 19 位，而 JS 处理数字类型时，只会处理到 16位。

解决方案，Integer id -> String id

  1. 跨域问题

协议、ip 地址、端口号任意一个不同，则产生跨域问题。

方法1：***@CrossOrgin***

方法2：网关 gateway，进行配置

  1. 413 问题

nginx 有上传文件的大小限制，在 nginx 中配置客户端上传文件大小。

  1. maven 加载 xml 问题

默认不加载 src/java 下的 xml 文件。

方法1：将 xml 文件复制到 target 目录下

方法2：

  • application.yml 中进行配置

  • 父 pom 文件中配置
